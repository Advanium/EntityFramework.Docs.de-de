<span data-ttu-id="2140d-122">Beachten Sie bei der Verwendung von Entitäten mit Selbstnachverfolgung Folgendes:</span><span class="sxs-lookup"><span data-stu-id="2140d-122">Consider the following when working with self-tracking entities:</span></span>

    > <span data-ttu-id="2140d-130">Wenn Objekte im Diagramm Eigenschaften mit datenbankgenerierten Werten (z.B. ## <a name="security-considerations"></a><span data-ttu-id="2140d-145">Sicherheitsüberlegungen</span><span class="sxs-lookup"><span data-stu-id="2140d-145">Security Considerations</span></span>  

<span data-ttu-id="2140d-146">Die folgenden Sicherheitsüberlegungen sollten Sie beim Arbeiten mit Entitäten mit Selbstnachverfolgung berücksichtigen:</span><span class="sxs-lookup"><span data-stu-id="2140d-146">The following security considerations should be taken into account when working with self-tracking entities:</span></span>  

- <span data-ttu-id="2140d-147">Ein Dienst sollte Anforderungen nicht vertrauen, Daten von einem nicht vertrauenswürdigen Client oder über einen nicht vertrauenswürdigen Kanal abzurufen oder zu aktualisieren.</span><span class="sxs-lookup"><span data-stu-id="2140d-147">A service should not trust requests to retrieve or update data from a non-trusted client or through a non-trusted channel.</span></span> <span data-ttu-id="2140d-148">Ein Client muss authentifiziert werden: Es sollte ein sicherer Kanal oder ein Nachrichtenumschlag verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="2140d-148">A client must be authenticated: a secure channel or message envelope should be used.</span></span> <span data-ttu-id="2140d-149">Clientanforderungen zum Aktualisieren oder Abrufen von Daten müssen überprüft werden, um sicherzustellen, dass sie den erwarteten und ordnungsgemäßen Änderungen für das jeweilige Szenario entsprechen.</span><span class="sxs-lookup"><span data-stu-id="2140d-149">Clients' requests to update or retrieve data must be validated to ensure they conform to expected and legitimate changes for the given scenario.</span></span>

- <span data-ttu-id="2140d-153">Um die Weitergabe von Ausnahmemeldungen mit vertraulichen Daten an die Clientebene zu verhindern, sollten Aufrufe von **ApplyChanges** und **SaveChanges** auf Serverebene von Ausnahmebehandlungscode umschlossen sein.</span><span class="sxs-lookup"><span data-stu-id="2140d-153">To avoid propagating exception messages that contain sensitive data to the client tier, calls to **ApplyChanges** and **SaveChanges** on the server tier should be wrapped in exception-handling code.</span></span>
